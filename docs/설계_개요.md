# RailPass - 기차표 자동 예매 웹 시스템 설계 개요

## 프로젝트 목표

기존 Windows PC 로컬 실행 방식의 Python CLI 예매 프로그램(`srtgo`)을 **웹 기반 서비스**로 전환하여,
어디서든 브라우저로 접속하여 SRT/KTX 기차표 자동 예매를 이용할 수 있는 시스템 구축.

## 기술 스택

### 프론트엔드
- **Next.js** (TypeScript) — 웹 페이지 및 UI
- **Zustand / React Query** — 전역 상태 관리 및 서버 상태 동기화
- **WebSocket / SSE** — 예매 실시간 상태 수신

### 백엔드 API
- **FastAPI** (Python) — REST API + WebSocket 처리
- **JWT** — 사용자 인증/인가
- **AES 암호화** — SRT/KTX 계정 및 카드 정보 암호화 저장

### 예매 워커
- **Python** (기존 srtgo 코드 재사용) — SRT/KTX 자동 예매 실행
- **BullMQ** (Redis 기반) — 예매 작업 큐잉

### 인프라
- **PostgreSQL** — 주 데이터베이스 (사용자, 예매 정보)
- **Redis** — 세션 캐시, 작업 큐

### 알림
- **Telegram Bot API** — 예매 성공 시 알림
- **Discord Webhook** — 예매 성공 시 알림

## 핵심 기능

### 1. 사용자 관리
- 이메일/비밀번호 기반 회원가입 및 로그인
- 사용자별 독립 설정 (다중 사용자 지원)

### 2. 계정 및 알림 설정
- SRT / KTX 계정 정보 (암호화 저장)
- 텔레그램 봇 토큰 + Chat ID 설정
- 디스코드 웹훅 URL 설정
- 카드 자동결제 정보 (암호화 저장)

### 3. 예매 설정 및 실행
- 열차 종류 선택 (SRT / KTX)
- 출발역 / 도착역 선택
- 출발 날짜 선택 (예매가능일 기준)
- **여러 시간대 복수 선택** (기존 CLI 개선)
- 승객 구성 (성인 / 어린이 / 경로우대 / 장애인)
- 좌석 유형 (일반실 우선 / 특실 우선 / 일반실만 / 특실만)
- 카드 자동결제 여부

### 4. 실시간 예매 현황
- WebSocket/SSE를 통한 실시간 상태 업데이트
- 시도 횟수 및 경과 시간 표시
- 예매 성공 시 즉시 페이지 업데이트

### 5. 예매 결과 및 관리
- 예매 내역 확인
- 미결제 예매 카드 결제
- 예매 취소 및 환불

## 예매 워커 동작 원리 (기존 코드 기반)

```
1. 사용자 예매 조건으로 작업 큐에 job 추가
2. 워커가 job 수신 → SRT/KTX 로그인
3. 열차 검색 반복:
   - 선택한 시간대의 열차 목록 조회
   - 좌석 가능 여부 확인
   - 가능하면 예매 시도
   - 예매 성공 → DB 저장 → 알림 발송 → 종료
   - 실패 시 gamma distribution(평균 1.25초) 간격으로 재시도
4. 오류 처리:
   - 세션 만료 → 자동 재로그인
   - NetFunnel 오류 → 세션 초기화 후 재시도
   - 연결 오류 → 재연결 후 재시도
```

## 보안 고려사항

- SRT/KTX 계정 비밀번호: AES-256 암호화 후 DB 저장
- 카드 정보: AES-256 암호화 후 DB 저장 (PCI DSS 준수 권장)
- API 통신: HTTPS (TLS 1.2+)
- 인증: JWT Access Token + Refresh Token 패턴
- 환경변수: 암호화 키는 `.env`로 관리 (Git 제외)
